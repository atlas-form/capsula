#### **7.2 登录与角色切换流程**

```typescript
// 登录流程
用户登录
  ↓
验证身份（调用Rust）
  ↓
获取用户角色（patient/doctor/hospital）
  ↓
加载对应角色的路由和界面
  ↓
显示角色专属功能

// 角色切换流程（如果用户有多个角色）
用户点击"切换角色"
  ↓
显示可用角色列表
  ↓
选择角色
  ↓
重新加载对应界面
  ↓
无需重新登录（token复用）
```

#### **7.3 权限控制实现**

```typescript
// 路由守卫
router.beforeEach((to, from, next) => {
  const userRole = getUserRole(); // patient/doctor/hospital
  const requiredRole = to.meta.role;

  if (requiredRole.includes(userRole)) {
    next(); // 允许访问
  } else {
    next('/unauthorized'); // 无权限
  }
});

// 组件级权限控制
<template>
  <div v-if="hasPermission('view_reports')">
    <!-- 只有有权限的用户才能看到 -->
  </div>
</template>

// Rust层权限验证
#[tauri::command]
async fn get_patient_data(user_id: String) -> Result<Data, Error> {
  // 验证用户是否有权限访问
  if !check_permission(user_id, "read_patient_data") {
    return Err(Error::Unauthorized);
  }
  // 返回数据
  Ok(data)
}
```
