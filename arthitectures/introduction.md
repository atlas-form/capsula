# 数据胶囊 v2（Signature-First Capsule）

> **一句话定义**
> 数据胶囊是一种：
> **由数据所有者签名的、可公开索引的事实声明容器，
> 原始数据仅作为可选的私有事实锚点存在。**

---

## 0️⃣ 设计原则（先立法）

这是 v2 和旧体系的**根本分水岭**：

### 原则 1：公开是默认，加密是例外

- 所有 **可被搜索、推理、组合** 的内容：
  - 必须是 **明文 + 可验证**

- 加密只用于：
  - 原始证据
  - 隐私字段
  - 法律风险数据

---

### 原则 2：签名即产权，验证即信任

- 谁能签名 **= 谁拥有数据主权**
- 谁能验证 **≠ 谁能访问原文**
- 智能体只依赖：
  - 哈希
  - 签名
  - 引用关系

---

### 原则 3：数据不是文件，是「声明（Claim）」

数据胶囊只关心三件事：

1. **声明了什么**
2. **谁声明的**
3. **是否可被独立验证**

---

## 1️⃣ 核心抽象（v2 新三元）

### ① Fact（事实）

> 不可再生、不可修改的锚点
> 可为原始文件、传感器记录、日志、影像等

- 可加密
- 不参与搜索
- 只参与 **哈希绑定**

---

### ② Claim（声明） ← v2 的核心单位

> 对事实的结构化主张
> **必须可公开、可索引、可签名**

例如：

```json
{
  "claim_type": "medical.blood.glucose",
  "value": ">7 mmol/L",
  "derived_from": "fact_hash",
  "method": "lab-test",
  "issued_at": "2026-01-01"
}
```

---

### ③ Proof（证明）

> 证明 Claim 不是编造的

Proof 可以是：

- 数字签名（默认）
- ZKP（可选）
- 引用链（Claim → Claim）

---

## 2️⃣ 新的分层结构（仍然 0 / 1 / 2，但语义已变）

### 🔹 0 阶：Fact Capsule（事实锚点层）

> **存在但不显眼**

```json
{
  "fact_id": "fact:sha256:...",
  "storage": {
    "type": "encrypted",
    "uri": "s3://bucket/raw.enc"
  },
  "hash": "sha256:...",
  "producer_signature": "sig(...)"
}
```

**特性**

- ❌ 不可搜索
- ❌ 不参与 Agent 推理
- ✅ 只用于：
  Claim 的可追溯性、法律仲裁、强验证

> 👉 在 v2 中，0 阶不再是“数据主体”，只是**证据仓库**

---

### 🔹 1 阶：Claim Capsule（事实声明层）⭐️ 核心

> **Agent 世界真正消费的对象**

```json
{
  "claim_id": "claim:uuid",
  "owner_id": "did:user:xxx",

  "claim": {
    "type": "medical.blood.glucose",
    "value": ">7 mmol/L",
    "unit": "mmol/L",
    "confidence": 0.98
  },

  "derived_from": {
    "fact_hash": "sha256:...",
    "method": "lab-test",
    "zkp": null
  },

  "issued_at": "2026-01-01T10:00:00Z",

  "signature": {
    "signer": "did:user:xxx",
    "alg": "ed25519",
    "sig": "base64..."
  }
}
```

**关键变化**

| 项目     | v1   | v2   |
| -------- | ---- | ---- |
| 默认状态 | 加密 | 明文 |
| 可搜索   | ❌   | ✅   |
| 主体     | 文件 | 声明 |
| 信任来源 | 解密 | 验证 |

---

### 🔹 2 阶：Claim Index Capsule（声明索引层）

> **为超级智能体设计的“公开事实目录”**

```json
{
  "index_id": "index:owner:xxx",

  "owner_id": "did:user:xxx",

  "claims": [
    {
      "type": "medical.blood.glucose",
      "claim_id": "claim:uuid"
    },
    {
      "type": "medical.blood.pressure",
      "claim_id": "claim:uuid2"
    }
  ],

  "index_hash": "sha256:...",
  "index_signature": "sig(owner)"
}
```

**特性**

- 全明文
- 高度可缓存
- 可被爬虫 / Agent 批量扫描
- 是“个人事实宇宙”的入口点

---

## 3️⃣ 加密在 v2 中的位置（被严格限制）

### 🔐 加密只允许出现在三处

1. **Fact 本体**
2. **敏感 Claim 字段**
3. **可撤销访问的附加证据**

例如：

```json
"private_payload": {
  "enc": "agead",
  "policy_ref": "acl://xxx"
}
```

> ⚠️ 加密内容 **永远不影响 Claim 的可验证性**

---

## 4️⃣ 权限、可控性、审计 —— 全部外移

### ❌ 胶囊内不再承载复杂控制逻辑

| 维度 | v1       | v2            |
| ---- | -------- | ------------- |
| 权限 | 内嵌     | 外部策略      |
| 审计 | 胶囊字段 | 外部事件流    |
| 撤销 | 重加密   | 版本 + 新声明 |

---

## 5️⃣ 新的“数据主权”定义（非常重要）

> **你不再拥有“谁能看数据”的绝对控制权，
> 你拥有的是：**

### 👉「**谁能代表这个事实发声**」

- 旧世界：

  > 谁有数据库 = 谁有权力

- v2 世界：

  > 谁有签名 = 谁有现实解释权

---

## 6️⃣ 这个设计天然支持什么未来？

### ✅ Agent 自由搜索

### ✅ 去平台化索引

### ✅ 用户级“事实空间”

### ✅ 声明组合 / 推理 / 反驳

### ✅ 数据真正从巨头回到个人

---

## 7️⃣ 一句残酷但真实的结论

> **v1 的数据胶囊，是“加密文件系统”
> v2 的数据胶囊，是“去中心化事实出版系统”**

你不是在做数据安全，你是在做：

> **AI 时代的个人事实主权协议**
